针对 `@electric-sql/pglite`（在浏览器/Node.js 中运行的嵌入式 PostgreSQL），我来提供几种实用的数据导入导出方案：

## 方案一：使用内置的 dump/restore 功能（推荐）

PGlite 支持 PostgreSQL 的 `pg_dump` 和 `pg_restore` 风格的 SQL 导出导入：

### 导出为 SQL 文件
```javascript
// scripts/export-sql.mjs
import { PGlite } from '@electric-sql/pglite';
import fs from 'fs';

const db = new PGlite('./data'); // 你的数据目录
await db.waitReady();

// 导出完整数据库（结构+数据）
const result = await db.query(`
  SELECT 
    'CREATE TABLE ' || tablename || ' AS ' || 
    (SELECT json_agg(t) FROM (SELECT * FROM tablename) t) 
  FROM pg_tables 
  WHERE schemaname = 'public'
`);

// 更简单的方式：使用 COPY 命令
const dumpResult = await db.query(`
  SELECT 
    'COPY ' || tablename || ' TO STDOUT WITH CSV HEADER;'
  FROM pg_tables 
  WHERE schemaname = 'public'
`);

// 实际导出
const tables = await db.query(`
  SELECT tablename FROM pg_tables WHERE schemaname = 'public'
`);

let sqlContent = '-- Export from PGlite\n\n';

for (const row of tables.rows) {
  const tableName = row.tablename;
  
  // 导出表结构
  const schema = await db.query(`
    SELECT 
      'CREATE TABLE IF NOT EXISTS ' || '${tableName}' || ' (' ||
      string_agg(column_name || ' ' || data_type, ', ') || ');'
    FROM information_schema.columns 
    WHERE table_name = '${tableName}'
    GROUP BY table_name
  `);
  sqlContent += schema.rows[0]?.string_agg + '\n\n';
  
  // 导出数据为 COPY 格式
  const data = await db.query(`COPY ${tableName} TO STDOUT WITH CSV HEADER`);
  sqlContent += data.rows.map(r => r.copy).join('\n') + '\n\n';
}

fs.writeFileSync('backup.sql', sqlContent);
console.log('✅ 导出完成: backup.sql');
```

### 从 SQL 文件导入
```javascript
// scripts/import-sql.mjs
import { PGlite } from '@electric-sql/pglite';
import fs from 'fs';

const db = new PGlite('./data');
await db.waitReady();

const sqlContent = fs.readFileSync('backup.sql', 'utf8');

// 分割并执行 SQL 语句
const statements = sqlContent
  .split(';')
  .filter(s => s.trim())
  .map(s => s.trim() + ';');

for (const stmt of statements) {
  try {
    await db.query(stmt);
  } catch (err) {
    console.warn('⚠️ 执行失败:', stmt, err.message);
  }
}

console.log('✅ 导入完成');
```

## 方案二：JSON 格式导入导出（更灵活）

### 导出为 JSON
```javascript
// scripts/export-json.mjs
import { PGlite } from '@electric-sql/pglite';
import fs from 'fs';

const db = new PGlite('./data');
await db.waitReady();

// 获取所有用户表
const tables = await db.query(`
  SELECT tablename 
  FROM pg_tables 
  WHERE schemaname = 'public'
    AND tablename NOT LIKE 'pg_%'
    AND tablename NOT LIKE 'sql_%'
`);

const exportData = {
  timestamp: new Date().toISOString(),
  version: '1.0.0',
  tables: {}
};

for (const row of tables.rows) {
  const tableName = row.tablename;
  const result = await db.query(`SELECT * FROM ${tableName}`);
  exportData.tables[tableName] = result.rows;
  
  // 导出表结构
  const schema = await db.query(`
    SELECT column_name, data_type, is_nullable
    FROM information_schema.columns 
    WHERE table_name = '${tableName}'
    ORDER BY ordinal_position
  `);
  exportData.tables[`${tableName}_schema`] = schema.rows;
}

fs.writeFileSync('backup.json', JSON.stringify(exportData, null, 2));
console.log('✅ 导出完成: backup.json');
console.log('📊 表数量:', Object.keys(exportData.tables).length / 2);
```

### 从 JSON 导入
```javascript
// scripts/import-json.mjs
import { PGlite } from '@electric-sql/pglite';
import fs from 'fs';

const db = new PGlite('./data');
await db.waitReady();

const importData = JSON.parse(fs.readFileSync('backup.json', 'utf8'));

// 先创建表结构
for (const [tableName, schema] of Object.entries(importData.tables)) {
  if (!tableName.endsWith('_schema')) continue;
  
  const realTable = tableName.replace('_schema', '');
  const columns = schema.map(col => 
    `${col.column_name} ${col.data_type} ${col.is_nullable === 'YES' ? '' : 'NOT NULL'}`
  ).join(', ');
  
  try {
    await db.query(`DROP TABLE IF EXISTS ${realTable} CASCADE`);
    await db.query(`CREATE TABLE ${realTable} (${columns})`);
    console.log(`✅ 创建表: ${realTable}`);
  } catch (err) {
    console.warn(`⚠️ 创建表失败 ${realTable}:`, err.message);
  }
}

// 导入数据
for (const [tableName, rows] of Object.entries(importData.tables)) {
  if (tableName.endsWith('_schema')) continue;
  if (rows.length === 0) continue;
  
  const columns = Object.keys(rows[0]);
  const placeholders = columns.map((_, i) => `$${i + 1}`).join(', ');
  const columnNames = columns.join(', ');
  
  for (const row of rows) {
    const values = columns.map(col => row[col]);
    try {
      await db.query(
        `INSERT INTO ${tableName} (${columnNames}) VALUES (${placeholders})`,
        values
      );
    } catch (err) {
      console.warn(`⚠️ 插入失败 ${tableName}:`, err.message);
    }
  }
  console.log(`✅ 导入数据: ${tableName} (${rows.length} 条)`);
}

console.log('🎉 导入完成');
```

## 方案三：使用 PGlite 的备份 API（最简洁）

PGlite 提供了 `dumpDataDir` 和 `loadDataDir` 方法：

```javascript
// scripts/backup-binary.mjs
import { PGlite } from '@electric-sql/pglite';
import fs from 'fs';

const db = new PGlite('./data');
await db.waitReady();

// 导出二进制数据目录
const dataDir = await db.dumpDataDir();
fs.writeFileSync('backup.pglite', dataDir);

console.log('✅ 备份完成: backup.pglite');
```

```javascript
// scripts/restore-binary.mjs
import { PGlite } from '@electric-sql/pglite';
import fs from 'fs';

const backupData = fs.readFileSync('backup.pglite');

// 从备份恢复
const db = new PGlite('./data', {
  dataDir: backupData
});

await db.waitReady();
console.log('✅ 恢复完成');
```

## 推荐使用场景

| 方案 | 适用场景 |
|------|---------|
| **SQL 导出** | 跨数据库迁移、版本控制 |
| **JSON 导出** | 数据可视化、导入其他系统 |
| **二进制备份** | 快速备份/恢复、迁移到另一台机器 |

## 注意事项

1. **数据目录位置**：确认你的 `PGlite` 实例使用的数据目录路径
2. **内存限制**：大数据集导出时注意内存使用
3. **编码问题**：确保导出的文件编码为 UTF-8
4. **事务处理**：导入大数据时考虑分批处理
